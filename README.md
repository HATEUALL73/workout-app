# workout-app

Мобильное приложение для отслеживания тренировок: упражнения, подходы, таймер отдыха, история и прогресс.

Приложение рассчитано на одного пользователя, работает без регистрации и бэкенда. Интерфейс выполнен на русском языке и ориентирован прежде всего на Android.

## Основные возможности

- выбор тренировочного дня;
- ввод веса и количества повторов для каждого подхода;
- сохранение незавершённых подходов в черновики;
- завершение упражнений с записью результата;
- история выполненных упражнений;
- таймер отдыха с готовыми интервалами, звуком и вибрацией;
- локальное хранение данных;
- экран прогресса со статистикой и графиком рабочего веса.

## Технологический стек

- Expo SDK 56;
- React Native 0.85.3;
- React 19.2.3;
- TypeScript 6 в строгом режиме;
- Expo Router;
- Expo SQLite;
- Expo Audio;
- Expo Haptics;
- React Native SVG;
- EAS Build.

## Требования

- Node.js версии, совместимой с Expo SDK 56 (не ниже 22.13.x);
- npm;
- Expo Go на Android для локального запуска;
- телефон и компьютер, подключённые к одной Wi-Fi сети.

На Windows используются команды `npm.cmd` и `npx.cmd`, поскольку PowerShell Execution Policy может блокировать запуск `npm.ps1` и `npx.ps1`.

## Установка

Откройте PowerShell и выполните:

```powershell
cd C:\Projects\workout-app
npm.cmd ci
```

## Локальный запуск

```powershell
npm.cmd start
```

После запуска:

1. Не закрывайте терминал — в нём работает сервер разработки Expo.
2. Откройте Expo Go на Android.
3. Отсканируйте QR-код, показанный в терминале.

Телефон и компьютер должны находиться в одной локальной сети. Для запуска через Expo Go отдельная нативная сборка не требуется. iOS Simulator доступен только на macOS.

## Проверки проекта

Проверка TypeScript:

```powershell
npx.cmd tsc --noEmit
```

Проверка совместимости установленных пакетов с текущей версией Expo:

```powershell
npx.cmd expo install --check
```

## Сборка

Профили EAS Build находятся в `eas.json`. Для `development`, `preview` и `production` настроены внутренние Android-сборки в формате APK. Команды сборки намеренно не приводятся: в `package.json` для них нет отдельных скриптов.

## Структура проекта

- `app/` — экраны и файловые маршруты Expo Router;
- `app/(tabs)/` — основные вкладки «Тренировка», «Таймер» и «Прогресс»;
- `components/` — переиспользуемые компоненты интерфейса, таймера и графика;
- `db.ts` — схема SQLite, миграции, исходные упражнения и функции работы с данными;
- `theme/` — цвета и тема навигации;
- `assets/` — иконки, изображения и звук окончания таймера;
- `format.ts` — форматирование дат, веса и пользовательского ввода;
- `CONTEXT.md` — продуктовые требования и исходная программа тренировок;
- `AGENTS.md` — инструкции по работе с проектом для Codex.

## Работа с Codex

Основные инструкции для Codex находятся в `AGENTS.md`. Локальный файл `.claude/settings.local.json` содержит настройки Claude Code, не является источником проектных требований и не должен коммититься.

## Примечания

- Все пользовательские данные хранятся локально на устройстве в SQLite.
- README описывает текущее состояние проекта.
- Не запускайте `npm audit fix` без отдельной проверки совместимости изменений с Expo и React Native.
